# blast_search.sh

A Bash script to BLAST a query FASTA file against all viable databases in a `blast_db` folder (as generated by `download_genbank.sh -b`). Results are written to a combined outfmt 6 file, a tab-delimited hit summary table, and a FASTA file of extracted hit sequences.

---

## Requirements

- `bash` 3.2+
- `blastn`, `blastp`, `blastx`, `tblastn` from [NCBI BLAST+](https://www.ncbi.nlm.nih.gov/books/NBK569861/)
- `python3`
- `awk`

The script checks all dependencies at startup and exits with a clear error if anything is missing.

---

## Usage

```bash
chmod +x blast_search.sh
./blast_search.sh -q query.fa -d ./blast_db
```

### Options

| Flag | Description | Default |
|------|-------------|---------|
| `-q` | Query FASTA file (nucleotide or protein) **(required)** | — |
| `-d` | Path to `blast_db` directory **(required)** | — |
| `-p` | BLAST program: `blastn`, `blastx`, `blastp`, `tblastn` | auto or required |
| `-i` | Minimum percent identity to report (0–100) | all hits |
| `-o` | Output file path | `<query_basename>_blast_results.txt` |
| `-e` | E-value threshold | `1e-5` |
| `-t` | Number of threads | `4` |
| `-h` | Show help | — |

---

## Program selection (`-p`)

The script auto-detects whether the query is nucleotide or protein by inspecting the sequence composition of the first record.

| Query type | Valid programs | When `-p` is required |
|------------|---------------|----------------------|
| Nucleotide | `blastn` (vs nucl dbs), `blastx` (vs prot dbs) | Both nucl and prot dbs present |
| Protein | `blastp` (vs prot dbs), `tblastn` (vs nucl dbs) | Both nucl and prot dbs present |

If only one program is valid (e.g. only nucl databases exist), it is selected automatically with no need for `-p`. If multiple programs are valid and `-p` is not given, the script exits with a descriptive error listing the options.

---

## Output files

All three output files share the same base name, derived from the output path (or the query filename if `-o` is not set).

### 1. BLAST results — `<base>_blast_results.txt`

A single outfmt 6 file with all results concatenated. Each database's results are separated by a comment header:

```
# ────────────────────────────────────────────────────────────
# Program:  blastp
# Database: GCF_017921755.1_prot
# Hits:     1
# Filter:   pident >= 90%
# ────────────────────────────────────────────────────────────
qseqid    sseqid    pident    length    ...
```

Columns: `qseqid sseqid pident length mismatch gapopen qstart qend sstart send evalue bitscore`

### 2. Hit summary table — `<base>_hit_summary.tsv`

A tab-delimited file with one row per database searched:

```
accession               hit
GCF_017921755.1_prot    yes
JABEBQ01_prot           yes
JABEBR01_prot           no
JABEBS01_prot           yes
```

Values are `yes`, `no`, or `error` (if the BLAST run itself failed).

### 3. Hit sequences — `<base>_hit_sequences.fa`

A FASTA file of sequences for all hits passing the e-value and identity thresholds, with `pident` and `evalue` appended to each header. Extraction method depends on the program:

**blastp / blastx** — protein sequences looked up directly from the `.faa` file by subject ID. NCBI pipe-format IDs (e.g. `ref|WP_013027259.1|`) are normalized automatically.

**blastn / tblastn** — subject regions extracted from the `.fna` file using the `sstart`/`send` coordinates reported by BLAST. Minus-strand hits (`sstart > send`) are automatically reverse-complemented. Headers include coordinates and strand:

```
>NZ_CP012345.1:45123-46891(plus) pident=98.50 evalue=1e-120
```

## Examples

```bash
# Auto-select program (only one type of db present)
./blast_search.sh -q rpoD.fa -d ./blast_db

# Specify program explicitly, filter by identity, custom output
./blast_search.sh -q rpoD.fa -d ./blast_db -p blastp -i 90 -e 1e-10 -o rpoD_results.txt

# Nucleotide query against nucl databases
./blast_search.sh -q my_gene.fa -d ./blast_db -p blastn -i 80
```

---

## Notes

- Databases are discovered automatically by scanning for `*_nucl.nin` and `*_prot.pin` index files in the `-d` directory
- The percent identity filter (`-i`) is applied after BLAST runs using `awk` on the outfmt 6 output — it does not affect which hits BLAST itself considers, only which are reported and extracted
- Hit sequences are deduplicated per database — if the same protein ID appears in multiple contigs, only the first occurrence is written to the FASTA
- Works with `blast_db` folders generated by `download_genbank.sh -b`
